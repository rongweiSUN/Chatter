## 功能链路

### 1. 短按热键 → 普通语音输入

```
录音 → ASR识别 → 技能管道 → 输出文本
```

- 先检查是否含唤醒词「随口说」→ 有则走语音 Agent（`gemini-3-flash`）
- 没有则看 `auto_run` 是否开启 → 开启则用 `gpt-oss-120b` 跑技能（口语过滤/结构化/自定义技能等）
- 最后输出处理后的文本

### 2. 长按热键 → 语音助手模式

```
录音 → ASR识别 → 语音Agent → (未处理则) → DeskClaw
```

- 用 `gemini-3-flash` 做意图识别 + function calling
- 如果是设置/技能管理指令（开关技能、添加技能等）→ 本地直接执行，完毕
- 其他所有请求 → 转发给 DeskClaw 处理

### 3. 选中文字 + 热键 → 指令模式

```
录音 → ASR识别 → 意图分类 ─┬─ 改写 → 用LLM改写选中文字
                           └─ 提问 → 用LLM回答问题
```

- 意图分类、改写、提问全部用 `gpt-oss-120b`
- 分类失败默认当「改写」处理

### 4. 回答窗口继续对话

```
用户输入 → DeskClaw
```

- 直接发给 DeskClaw，不经过本地 LLM

## 配置加载优先级

1. `~/Library/Application Support/VoiceInput/settings.json`（用户本地配置）
2. `default_settings.json`（内置默认，打包在 app bundle 内）
3. 代码中的 dataclass 默认值

如果用户本地配置存在但 ASR 凭证或 LLM provider 为空，会自动从内置默认补全（见 `Settings.load()`）。

---

## 一句话总结

- **`gemini-3-flash`** 只用在语音助手 Agent（管理设置和技能）
- **`gpt-oss-120b`** 用在所有文本处理（技能管道、意图分类、改写、提问）
- **DeskClaw** 处理语音助手中非设置管理的通用请求
- **ASR** 固定火山引擎，不可切换
